<h1>Основные этапы анализа<h1>

1. Загрузка и первоначальный осмотр данных
   
Проект начинается с загрузки CSV-файла `````ncr_ride_bookings.csv````` с обработкой возможных ошибок:

`````
try:
    df = pd.read_csv("ncr_ride_bookings.csv")
    print("Файл загружен успешно!")
    print(f"Размер данных: {df.shape[0]} строк, {df.shape[1]} столбцов")
except FileNotFoundError:
    print("Файл 'ncr_ride_bookings.csv' не найден")
    exit()
`````
 Использование блока try-except обеспечивает стабильность работы программы при отсутствии файла или других ошибках ввода-вывода.

2. Исследование структуры данных
   
После успешной загрузки выполняется анализ структуры данных:

`````
print(f"Реальные названия столбцов ({len(df.columns)}):")
for i, col in enumerate(df.columns, 1):
    print(f"  {i}. {col}")
`````
Автоматическое определение реальных названий столбцов делает код универсальным и не зависящим от конкретного формата исходных данных.

3. Анализ качества данных
   
Проверка пропущенных значений и статистический анализ:

`````
missing = df.isnull().sum()
for col, count in missing.items():
    if count > 0:
        print(f"  {col}: {count} пропусков")
`````
Детальный анализ пропущенных значений помогает оценить качество данных и необходимость их предобработки.

4. Интеллектуальная фильтрация данных

 Проект реализует умный поиск столбцов по ключевым словам:

 `````
status_columns = [col for col in df.columns if 'status' in col.lower() or 'Status' in col]
vehicle_columns = [col for col in df.columns if 'vehicle' in col.lower() or 'Vehicle' in col]
date_columns = [col for col in df.columns if 'date' in col.lower() or 'time' in col.lower()]
 `````
Такой подход позволяет работать с различными форматами данных без необходимости ручного указания названий столбцов.

5. Фильтрация по статусам

Автоматическое определение и фильтрация отмененных бронирований:

 `````
cancel_statuses = [s for s in unique_statuses if 'cancel' in str(s).lower()]
if cancel_statuses:
    cancel_status = cancel_statuses[0]
    cancelled = df[df[status_col] == cancel_status]
 `````
Алгоритм самостоятельно находит статусы, содержащие слово "cancel", что делает анализ гибким к различным формулировкам в данных.
